6、数组的扩展

[TOC]

¶数组扩展

¶方法

¶转为真正数组

  • Array.from(数据结构,function(){return;},对象)

  • 将两类对象转为真正的数组 类数组的对象 可遍历的对象(Iterable) 数组返回数组

  • 扩展运算符也能将具有Iterator接口的数据结构转换为数组(类数组对象不行)

  • 任何有length属性的对象都能通过Array.from()转为数组

¶第二个参数
  • Array.from还可以接受第二个参数 作用类似于数组的map方法 用来对每个元素进行处理，将处理后的值放入返回的数组
¶第三个参数
  • 如果在map函数中使用了this 可以传入第三个参数(绑定this) 传入一个对象

¶将一组值转换为数组

  • Array.of(值1,值2,值3)

¶实例方法

¶将指定位置的成员复制到其他位置（会覆盖原有成员）

  • 实例.copyWithin(target,start,end) 三个参数都是index表示

  • target 从该位置开始替换数据

  • start 从该位置读取数据 默认为0 负值表示倒数 包括

  • end 到改位置停止读取数据 默认为length 负值表示倒数 不包括

  • 三个参数都是数值 如果不是 自动转为数值

  • 从start位置开始读取数据到end为止 将数据覆盖到从target开始的位置

¶找出第一个符合条件的数值成员

  • 实例.find(function(value,index,arr){return;},对象) 可以识别NaN

  • 直到找出第一个返回值为true的成员 然后返回改成员 否则返回undefined

  • 回调函数的参数 value 当前的成员 index 该成员对应的索引值 arr 数组本身

  • 可以接收第二个参数 绑定this

  • 传统的indexof方法不能识别NaN

¶找出第一个符合条件的数值成员的索引

  • 实例.findIndex(function(value,index,arr){return;},对象) 可以识别NaN

  • 直到找出第一个返回值为true的成员 然后返回改成员的索引值 否则返回-1

  • 回调函数的参数 value 当前的成员 index 该成员对应的索引值 arr 数组本身

  • 可以接收第二个参数 绑定this

¶使用给定值 填充数值

  • 实例.fill(value,start,end)

  • value 填充的值

  • start 起始位置 包括 默认0

  • end 结束位置 不包括 默认length

  • 将value填充到start到end的位置上(覆盖)

¶遍历键

¶遍历值

¶遍历键值对

¶检查数值是否包含给定的值（返回bool）es7

  • 实例.includes(数据结构,start)

  • start 表示搜索的起始位置 负值表示倒数位置 大于数组长度会重置为0

  • 与map和set数据结构中has方法的区别

    • Map结构的has方法，是用来查找键名的，比如Map.prototype.has(key)、WeakMap.prototype.has(key)、Reflect.has(target, propertyKey)。
    • Set结构的has方法，是用来查找值的，比如Set.prototype.has(value)、WeakSet.prototype.has(value)。

¶数组的空位

  • 数组的空位指数组的某一个位置没有任何值 Array(3) //[,,]
  • 空位不是undefined 空位没有任何值

¶es5对空位的处理

  • forEach() filter() every() 和some()都会跳过空位

  • map()会跳过空位 但会保留这个值

  • join()和toString()会将空位视为undefined 而undefined和null会被处理成空字符串

¶es6对于空位处理

  • es6会将空位转为undefined

[TOC]

¶数值扩展

¶二进制和八进制表示法

  • 使用Number()方法可以将0b和0o前缀的字符串数值转为十进制

¶方法

¶检查一个数值是否有限

  • Number.isFinite(数值)

¶检查是否为NaN

  • Number.isNaN(数值)
  • 它们与传统的全局方法isFinite()和isNaN()的区别在于，传统方法先调用Number()将非数值的值转为数值，再进行判断，而这两个新方法只对数值有效，非数值一律返回false。

¶解析字符串 返回整数

  • Number.parseInt(string,radix)
  • radix 可选 要解析的数字基数(2-36之间 即进制) 如果为0或省略 以10解析
  • 如果以0x或0X开头 以16解析 如果参数小于2或大于36 返回NaN
  • 如果string为八进制字符串 返回的可能为10进制也可能为8进制

¶解析字符串 返回浮点数

  • Number.parseFloat(string)
  • 如果字符串首字符为数字(可以是空格) 对字符串解析 直到末端 不是则返回NaN
  • 将es5的方法转移到Number对象上 使语言逐步模块化

¶判断一个值是否为整数

  • Number.isInteger() 整数.0会被当成整数

¶Number.EPSILON

  • 引入一个极小的常量
  • 浮动数计算是不准确的 如果误差小于Number.EPSILON 则认为得到正确结果

¶安全整数

  • JavaScript能够准确表示的整数范围在-2^53到2^53之间（不含两个端点）超过这个范围，无法精确表示这个值

  • 引入Number.MAX_SAFE_INTEGER 和 Number.MIN_SAFE_INTEGER 表示这个范围得上下限

¶判断一个数是否在极限范围里

  • Number.isSafeInteger()

¶math对象扩展

¶去除小数部分 返回整数部分

  • Math.trunc()

¶判断一个数是正数、负数、0

  • Math.sign() 正数返回 +1 负数返回 -1 0返回0 -0返回-0 其他值返回NaN

¶计算立方根

  • Math.cbrt()

¶promise对象

  • 保存着某个未来才会结束的事件（通常是异步操作）的结果 从它可以获取异步操作的消息
  • promise对象状态不受外界影响（三种状态 pending进行中 fulfilled已成功 rejected已失败）
  • 状态一旦改变 就不会再变 任何时候都可以得到这个结果
  • 无法取消promise 一旦新建就会立即执行 如果不设置回调函数 错误不会反应到外部

1、let和const命令

[TOC]

¶let和const

¶let

  • let 变量=值

  • 只在当前作用域内有效(可用于for循环) 即{} 这样的代码块

  • 不存在变量提升(提前声明)
  • 存在暂停性死区(在变量声明前不能使用该变量)
  • 不允许重复声明

¶使用var的风险

  • 内层变量可能会覆盖外层变量 (提前声明)

  • 用于循环计数的变量泄露为全局变量

¶es6的块级作用域

  • 只要是{}就是一个块级作用域

  • 可以任意嵌套 内层作用域可以定义外层作用域的同名变量

  • 允许在块级作用域中声明函数(es5前不允许 在严格模式下同样无效)

  • 本质上es6的块级作用域是一个语句 将多个操作封装在一起(操作即作用域中的执行语句) 没有返回值(所有外层无法访问内层)

  • do{} 返回块级作用域最后执行的表达式的值 (不能使用)

¶const

  • const 变量=值

  • 声明一个只读的常量 值不能改变

  • const一旦声明必须初始化(赋值)

  • 不存在变量提升(提前声明)

  • 存在暂停性死区(在变量声明前不能使用该变量)

  • 不能重复声明

  • const只是保证变量名指向的地址不变 但该地址的数据可能改变(使用const定义对象 该对象可以增删属性)

  • 将对象冻结 const 变量=Object.freeze( {} ) 不能增删属性

¶顶层对象的属性

  • 顶层对象在浏览器环境指的是window对象 在node指的是global对象

  • 在es5中 顶层对象的属性与全局变量是等价的

  • 在es6中 var命令和function命令声明的全局变量依然是顶层对象的属性 let const class命令声明的全局变量 不属于顶层对象的属性

¶global对象

  • 同一段代码不能在任何环境下都能拿到顶层对象
  • 引入global 在所有环境中都可以通过global获取顶层对象
